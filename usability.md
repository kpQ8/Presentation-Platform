# Usability

## Navigation
- Navigation section is persistent across all sections to allow users to navigate each section
- Thread list sidebar is also always visible to allow users to quickly switch back to threads
- Certain elements may be clicked for further navigation, such as thread author's to view their profile, or threads in this profile page to navigate to a thread.

## Visual Feedback
- Buttons have a clear 3D press effect to provide a tactile feeling
- Like and watch buttons change colour to provide a visualisation of the current state
- Clickable items have some form of hover effect to highlight that they are clickable, either the cursor will change or the button will change colour

## Error Handling
- All API errors are clearly shown at the top of the screen to show the user what error has occured
- The error has a close button to ensure it is dismissable
- Form validation gives immediate feedback

## Forms and Inputs
- Edit forms are pre-populated to show the user the current value of what they are changing
- Comment input boxes are placed appopriately for when there are or aren't any existing comments

## Comments
- Comments are indented to show whether it is a reply or not
- Sorted in reverse chronological order

# Mobile
- Layout switches to support differing screen sizes
